# SECP3623 - Database Programming

## 📌 Course Overview
This course focuses on advanced database programming concepts involving both relational and non-relational database systems. It covers SQL for structured data management as well as MongoDB for NoSQL document-based databases. The course also introduces important database concurrency control techniques such as Two-Phase Locking (2PL) to ensure data consistency and integrity in multi-user environments.

---

## 🎯 Learning Outcomes
- Write complex SQL queries for data manipulation and retrieval
- Design and manage relational database systems
- Understand NoSQL databases using MongoDB
- Apply concurrency control techniques in databases
- Analyze transaction management and isolation levels

---

## 📚 Topics Covered

### 🗄️ SQL (Relational Database)
- Advanced SELECT queries
- JOIN operations (INNER, LEFT, RIGHT, FULL)
- Subqueries and nested queries
- Aggregation functions (SUM, COUNT, AVG)
- Indexing and query optimization
- Views and stored procedures

---

### 🍃 MongoDB (NoSQL Database)
- Document-based data model (JSON/BSON)
- Collections and documents
- CRUD operations in MongoDB
- Aggregation pipeline
- Schema design in NoSQL
- Comparison between SQL vs NoSQL

---

### 🔐 Transaction Management
- ACID properties (Atomicity, Consistency, Isolation, Durability)
- Transaction lifecycle
- Commit and rollback operations

---

### 🧩 Concurrency Control
- Two-Phase Locking (2PL)
- Lock-based protocols
- Shared vs exclusive locks
- Deadlock detection and prevention
- Isolation levels (Read Uncommitted, Read Committed, Repeatable Read, Serializable)

## 💡 Key Technical Understanding

Through this course, I gained a strong foundation in both relational and non-relational database systems. SQL provided a structured way to manage and query data efficiently, while MongoDB introduced flexibility in handling unstructured and semi-structured data. Understanding both systems helped me appreciate how modern applications often use hybrid database architectures.

A particularly important concept learned in this course was transaction management and concurrency control using Two-Phase Locking (2PL). This ensured that multiple users accessing the database simultaneously do not cause inconsistency or data corruption. I also learned how isolation levels affect performance and data integrity in real-world systems.

---

## 📖 Reflection
This course significantly enhanced my understanding of how modern database systems operate under both structured and unstructured environments. Initially, I found the transition between SQL and MongoDB challenging because both systems use completely different data models. However, through hands-on exercises, I developed a clearer understanding of when to use relational databases versus NoSQL databases depending on application requirements.

One of the most challenging yet important topics was concurrency control, especially Two-Phase Locking (2PL). Understanding how databases handle multiple transactions simultaneously helped me realize the importance of data integrity in multi-user systems such as banking applications, e-commerce platforms, and cloud-based systems.

Overall, this course strengthened my database engineering skills and provided a strong foundation for backend development and data engineering pipelines. It also improved my ability to design scalable and reliable database systems that can handle real-world workloads efficiently.
