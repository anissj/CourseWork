# SECP3843 - Special Topic in Data Engineering (WBL)

## 📌 Course Overview
This course focuses on advanced topics in data engineering, particularly in modern data architecture and large-scale data management systems. It covers the design and implementation of data ecosystems such as data lakes, data warehouses, data marts, and lakehouse architectures. The subject emphasizes how data is collected, stored, processed, and transformed into meaningful insights for analytical and business purposes in real-world industry environments.

---

## 🎯 Learning Outcomes
- Understand modern data architecture concepts
- Differentiate between data warehouse, data lake, and data lakehouse
- Design scalable data storage and processing systems
- Apply data modeling techniques for analytics
- Analyze real-world data engineering architectures

---

## 📚 Topics Covered

### 🏗️ Data Architecture
- Traditional vs modern data architecture
- Batch vs real-time data systems
- ETL vs ELT pipelines

### 🗄️ Data Warehouse
- Structured data storage
- OLAP systems
- Star schema and snowflake schema
- Business intelligence integration

### 🌊 Data Lake
- Raw and unstructured data storage
- Schema-on-read approach
- Big data storage systems (Hadoop ecosystem concepts)

### 🧱 Data Lakehouse
- Hybrid architecture (Data lake + Data warehouse)
- Unified analytics platform
- Delta Lake / modern lakehouse concepts

### 📊 Data Mart
- Department-level data storage
- Subject-oriented datasets
- Faster query performance for business units

### 🔄 Data Pipelines
- ETL (Extract, Transform, Load)
- ELT (Extract, Load, Transform)
- Data ingestion and transformation workflows

### ☁️ Modern Data Engineering Concepts
- Scalable distributed systems
- Cloud-based data platforms
- Data governance and quality
- Metadata management

## 💡 Key Industry Understanding

Through this course, I gained exposure to how modern organizations manage massive volumes of data using structured and unstructured systems. I learned how data architecture plays a critical role in ensuring scalability, efficiency, and accessibility of data for analytics and decision-making. The subject also highlighted the evolution from traditional data warehouses to modern lakehouse architectures, which combine the strengths of both structured and unstructured data systems.

---

## 📖 Reflection
This course significantly deepened my understanding of modern data engineering concepts and how large-scale data systems are designed in real-world industries. I learned that data is not simply stored, but carefully architected to ensure efficiency, scalability, and usability across different business functions. The distinction between data warehouse, data lake, data mart, and data lakehouse helped me understand how organizations structure their data ecosystems based on different analytical needs.

One of the most important insights from this subject was the evolution of data architecture from traditional centralized systems to modern distributed and hybrid models such as the data lakehouse. This concept is particularly relevant in today’s industry where companies deal with both structured and unstructured data at massive scale. I also gained a better understanding of ETL and ELT processes, which are essential in building efficient data pipelines.

From a personal perspective, this course strengthened my identity as a data engineering student because it connects directly with real-world industry practices. It improved my ability to think at a system architecture level rather than just focusing on coding. Overall, this subject provided a strong foundation for understanding how modern data-driven organizations operate and manage their data assets.
